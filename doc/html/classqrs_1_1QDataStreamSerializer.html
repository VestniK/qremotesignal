Detailed Description

Serializer based on QDataStream standard Qt class.

This class serializes internal library remote call representation into a binary message using Qt4 QDataStream class. serialize and deserialize funtions are reentrant and thus you can use single instance of this class in different threads.

Order and format of serialization of qrs::Message class is the following:

  1. Message type as qint8 number.
  2. Error type as qint8 number.
  3. Error string (even if empty) as QString.
  4. Service name as QString.
  5. Method name as QString.
  6. Renote call parameters as QVariantMap.

You can create and manage instances of this class manually but it's more convivient to use predefined macroses which allows you to have one global instance of this class.

Note
QDataStream protocol version older then QDataStream::Qt_3_3 can't pass tests from the serializers test suit. Ussage of such versions is depricated.
